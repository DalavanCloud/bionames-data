        Hey,

        This looks like an interesting project. I would handle the problem as following:

        1) High threshold ( so only black will stay)
        2) Dilate the image (the barcode will become one component)
        3) Determine the connected components and keep the largest component -> the barcode.
        4) If you look clearly the 2 codes you need, have the same y-distance relative to the barcode.
        5) Using the ratio (of the barcode) you can calculate the positions of both codes, although you can also calculate the approximate height and width of those characters (in function of the ratio of the barcode).
        6) After you calculated the positions of both codes, you can pass them both to a different tesseract operation.

        I hope this can help you.

        Cédric

            I already applied the links in your previous post, the method I propose is based on the images you've shared there.

            OCR will detect everything what it thinks to be a valuable character. I can't imagine how your "current" method can accomplish without any kind of pre-processing. Tesseract will detect all possible characters. And so all the other numbers, and it will also use the other characters to improve the end result. Tesseract doesn't specify any option to detect characters in a certain region. "You have to use an digitale image processing library for that"

            If you would use a vision library "OpenCV" or any other kind of image processing library, you can filter out the lottery numbers + draw date.

            An other and very simple method is to split the images in parts. Cut out a static region for the lottery numbers and either for the draw date. "This can also be accomplished by OpenCV".

              Hi Burak,

              I used Tesseract 3.01 in this tutorial, forgot to mention this "important" information. I didn't test the training procedure for 3.02.

              The shapetables are optional in Tesseract 3.01, they are computed from the shapeclustering step (I didn't used this). On Tesseract SVN you can find:
              " (Optional) When present, a shapetable is an extra layer between the character
              classifier and the word recognizer that allows the character classifier to
              return a collection of unichar ids and fonts instead of a single unichar-id
              and font."

              Microfeat is idd not used anymore..

              The procedure I've described seems to be enough. I'm using a training file atm for my thesis and am getting accurate results. It should work with 3.01, I will try to update this post with significant changes for version 3.02

              I hope I answered some of your questions, a lot of information can be found on the Tesseract official website (https://code.google.com/p/tess...

              Kindly Regards
              Cédric
